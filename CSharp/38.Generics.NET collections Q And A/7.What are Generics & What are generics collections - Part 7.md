
# Introduction to Generics in C# (.NET Collections)

## Overview

Earlier .NET collections such as **Array**, **ArrayList**, and **Hashtable** each have strengths and weaknesses.

| Collection | Strength | Weakness |
|------------|----------|----------|
| Array | Fast, strongly typed | Fixed size |
| ArrayList | Dynamic size | Stores objects, boxing/unboxing |
| Hashtable | Key-value lookup | Stores objects, boxing/unboxing |

As developers, we want a collection that is:

- Strongly typed
- Dynamic (resizable)
- High performance
- Easy to maintain

This is where **Generics** come into the picture.

> **Important:** Generics and Generic Collections are related but different concepts.
>
> - **Generics** are a language feature.
> - **Generic Collections** are collections built using generics.

---

# What Are Generics?

Generics allow you to **separate business logic from the data type**.

Instead of writing the same code for `int`, `string`, `bool`, etc., you write the logic once and specify the data type when using the class or method.

This improves:

- Code reuse
- Maintainability
- Type safety
- Performance

---

# Why Were Generics Introduced?

Without generics, you might write separate methods:

```csharp
CompareInt(int x, int y)

CompareString(string x, string y)

CompareDecimal(decimal x, decimal y)
```

This duplicates logic.

Generics let you write the comparison logic once.

---

# Generic Class Example

```csharp
public class Check<T>
{
    public bool Compare(T x, T y)
    {
        return x.Equals(y);
    }
}
```

Here, `T` represents an unknown type that will be specified later.

---

# Using a Generic Class with int

```csharp
Check<int> intCheck = new Check<int>();

bool result = intCheck.Compare(10, 10);

Console.WriteLine(result);
```

Output

```
True
```

---

# Using a Generic Class with string

```csharp
Check<string> stringCheck = new Check<string>();

bool result = stringCheck.Compare("Shiv", "Raj");

Console.WriteLine(result);
```

Output

```
False
```

---

# How Generics Work

When you write:

```csharp
Check<int> check = new Check<int>();
```

The compiler treats `T` as `int`.

```
Before Compilation

Check<T>

        │

        ▼

Runtime / Compile-Time Type Binding

        │

        ▼

Check<int>
```

Similarly,

```csharp
Check<string>
```

treats `T` as `string`.

The same logic works for multiple data types.

---

# Strong Typing

Once a type is specified, the class becomes strongly typed.

```csharp
Check<int> check = new Check<int>();

check.Compare(10, 20);      // Valid

check.Compare("A", "B");    // Compile-time error
```

No casting is required.

---

# Advantages of Generics

## 1. Type Safety

The compiler ensures that only the specified type is used.

---

## 2. Better Performance

No boxing or unboxing for value types.

---

## 3. Code Reusability

One implementation works with many data types.

---

## 4. Easy Maintenance

Business logic is written once and reused.

---

# Generics vs Object-Based Code

Without Generics

```csharp
public bool Compare(object x, object y)
{
    return x.Equals(y);
}
```

Problems:

- No compile-time type checking
- Requires casting
- Boxing/unboxing for value types

---

With Generics

```csharp
public bool Compare<T>(T x, T y)
{
    return x.Equals(y);
}
```

Benefits:

- Type-safe
- Better performance
- No casting
- Cleaner code

---

# Relationship Between Generics and Generic Collections

Generics are the foundation for generic collections.

Examples of generic collections include:

- `List<T>`
- `Dictionary<TKey, TValue>`
- `Queue<T>`
- `Stack<T>`
- `HashSet<T>`

These collections combine:

- Dynamic sizing
- Strong typing
- High performance

---

# Generic Collections Namespace

```csharp
using System.Collections.Generic;
```
