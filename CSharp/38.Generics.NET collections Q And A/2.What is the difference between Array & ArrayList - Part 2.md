
# Array vs ArrayList in C# (.NET Collections)

## Overview

This topic explains the differences between **Array** and **ArrayList**, the shortcomings of arrays, how ArrayList solves them, and when each should be used.

---

# What is an Array?

An **Array** is a fixed-size, strongly typed collection that stores elements of the same data type.

Example:

```csharp
string[] names = new string[10];
```

Important characteristics:

- Fixed size
- Strongly typed
- Fast access using indexes
- Better performance than ArrayList

---

# Example – Adding Data to an Array

```csharp
string[] arr = new string[10];

for (int i = 0; i < arr.Length; i++)
{
    if (arr[i] == null)
    {
        arr[i] = textBox1.Text;
        break;
    }
}
```

To display:

```csharp
listBox1.DataSource = null;
listBox1.DataSource = arr;
```

---

# Problems with Arrays

## 1. Fixed Size

```csharp
string[] names = new string[10];
```

Once created, the size cannot be increased.

To resize:

1. Create a new array.
2. Copy old values.
3. Replace the old array.

---

## 2. Strongly Typed

```csharp
int[] numbers = new int[5];
```

Only integers can be stored.

Invalid:

```csharp
numbers[0] = "Hello"; // Compile-time error
```

---

## 3. More Coding Required

To insert a value, you often need to:

- Find an empty slot.
- Loop through the array.
- Assign the value manually.

This makes the code verbose.

---

# What is ArrayList?

`ArrayList` is a dynamically resizable collection available in `System.Collections`.

```csharp
using System.Collections;

ArrayList list = new ArrayList();
```

---

# Adding Elements

```csharp
ArrayList list = new ArrayList();

list.Add("India");
list.Add("USA");
list.Add("Nepal");
```

No need to specify a size.

---

# Binding

```csharp
listBox1.DataSource = list;
```

---

# Advantages of ArrayList

## 1. Dynamic Size

```csharp
ArrayList list = new ArrayList();

list.Add("A");
list.Add("B");
list.Add("C");
```

The collection automatically grows.

---

## 2. Can Store Different Types

```csharp
ArrayList list = new ArrayList();

list.Add(100);
list.Add("Hello");
list.Add(DateTime.Now);
list.Add(new Customer());
```

---

## 3. Simple Methods

Common methods:

```csharp
list.Add(item);
list.Remove(item);
list.RemoveAt(index);
list.Clear();
list.Insert(index, item);
```

No manual loops are required for insertion.

---

# Array vs ArrayList

| Feature | Array | ArrayList |
|--------|--------|-----------|
| Size | Fixed | Dynamic |
| Type Safety | Strongly typed | Stores any object |
| Performance | Faster | Slightly slower |
| Memory | Lower | Higher |
| Resize | Manual | Automatic |
| Add Method | No | Yes |
| Remove Method | No | Yes |

---

# Performance

### Array

- Better performance
- Less memory overhead
- Compile-time type checking

### ArrayList

- Slightly slower
- Stores objects
- Boxing/Unboxing for value types

---

# When Should You Use Array?

Use Array when:

- Size is known.
- Same data type.
- Performance is critical.
- Memory efficiency matters.

Examples:

- Months of a year
- Days of a week
- Fixed sensor values

---

# When Should You Use ArrayList?

Use ArrayList when:

- Number of items is unknown.
- Frequent additions/removals.
- Mixed object types (legacy code).

---

# Modern Recommendation

In modern C#, **avoid ArrayList** for new applications.

Instead, use **List<T>**.

```csharp
List<string> countries = new List<string>();

countries.Add("India");
countries.Add("USA");
countries.Add("Nepal");
```

Benefits:

- Dynamic size
- Type safe
- Better performance than ArrayList
- No boxing/unboxing

---

# Interview Questions

## Why is Array fixed?

Because memory is allocated at creation time.

---

## Why is ArrayList slower?

It stores elements as `object`, causing boxing/unboxing for value types.

---

## Why is List<T> preferred over ArrayList?

- Type safety
- Better performance
- Generic
- No boxing/unboxing

---

## Can ArrayList store multiple data types?

Yes.

```csharp
ArrayList list = new ArrayList();

list.Add(100);
list.Add("Hello");
list.Add(true);
```

---

# Key Takeaways

- Array is fixed-size and strongly typed.
- ArrayList is dynamically resizable.
- ArrayList provides Add(), Remove(), Insert(), and Clear().
- ArrayList can store different data types.
- Arrays are faster.
- For modern .NET development, **List<T>** is the recommended replacement for ArrayList.
