
# Array vs ArrayList Performance in C#

## Overview

Although **ArrayList** provides dynamic sizing and a simpler programming model, **Array** is still faster in many scenarios.

This note explains **why arrays outperform ArrayList**, demonstrates a performance comparison using `Stopwatch`, and discusses when to choose each.

---

# Why Are Arrays Faster?

Arrays are:

- Fixed-size
- Strongly typed
- Stored in contiguous memory
- No boxing/unboxing for value types

These characteristics make arrays faster than `ArrayList`.

---

# Why Is ArrayList Slower?

`ArrayList`:

- Stores elements as `object`
- Performs boxing/unboxing for value types
- Performs automatic resizing when needed
- Has additional runtime overhead

---

# Performance Test

## Goal

Compare:

1. Insert 1000 items into an Array
2. Iterate through the Array
3. Insert 1000 items into an ArrayList
4. Iterate through the ArrayList
5. Measure execution time using `Stopwatch`

---

# Namespaces

```csharp
using System.Collections;
using System.Diagnostics;
```

---

# Create Collections

```csharp
string[] array = new string[1000];

ArrayList arrayList = new ArrayList();
```

---

# Measure Array Performance

```csharp
Stopwatch sw = new Stopwatch();

sw.Start();

for (int i = 0; i < array.Length; i++)
{
    array[i] = DateTime.Now.Second.ToString();
}

foreach (string item in array)
{
    var value = item;
}

sw.Stop();

Console.WriteLine($"Array: {sw.ElapsedTicks}");
```

---

# Measure ArrayList Performance

```csharp
sw.Reset();
sw.Start();

for (int i = 0; i < 1000; i++)
{
    arrayList.Add(DateTime.Now.Second.ToString());
}

foreach (string item in arrayList)
{
    var value = item;
}

sw.Stop();

Console.WriteLine($"ArrayList: {sw.ElapsedTicks}");
```

---

# Typical Result

```text
Array      : 120 ticks
ArrayList  : 180 ticks
```

The exact values vary by machine, but **Array consistently completes faster**.

---

# Why Arrays Win

## Strong Typing

```csharp
int[] numbers = new int[100];
```

The compiler knows the exact data type.

---

## Fixed Memory

Memory is allocated once.

No resizing is required.

---

## No Boxing / Unboxing

Arrays store value types directly.

ArrayList stores value types as `object`, which causes:

- Boxing during insertion
- Unboxing during retrieval

---

# Array vs ArrayList Performance

| Feature | Array | ArrayList |
|---------|-------|-----------|
| Speed | Faster | Slower |
| Memory | Less | More |
| Type Safety | Yes | No |
| Boxing/Unboxing | No | Yes (value types) |
| Resize | Manual | Automatic |

---

# When to Use Array

Use Array when:

- Size is fixed
- High performance is required
- Data type is known
- Memory efficiency is important

Examples:

- Image pixels
- Mathematical calculations
- Fixed configuration values

---

# When to Use ArrayList

Use ArrayList only for:

- Legacy .NET Framework applications
- Code that specifically requires `ArrayList`

For new applications, prefer `List<T>`.

---

# Modern Recommendation

Instead of `ArrayList`, use:

```csharp
List<string> list = new List<string>();

list.Add("India");
list.Add("USA");
```

Benefits:

- Dynamic size
- Type safe
- Faster than ArrayList
- No boxing/unboxing

# Interview Questions

## Why are arrays faster than ArrayList?

Because arrays are strongly typed, fixed-size, and avoid boxing/unboxing.

---

## Which class is used to measure execution time?

`Stopwatch` from `System.Diagnostics`.

---

## Which property returns execution time?

```csharp
stopwatch.ElapsedTicks
```

---

## Should ArrayList be used in modern .NET?

Generally **No**. Use `List<T>` unless maintaining legacy code.

---

# Key Takeaways

- Arrays offer the best performance for fixed-size collections.
- ArrayList provides flexibility but incurs additional overhead.
- `Stopwatch` is the standard tool for benchmarking execution time.
- For modern C# development, **List<T>** is preferred over `ArrayList`.
