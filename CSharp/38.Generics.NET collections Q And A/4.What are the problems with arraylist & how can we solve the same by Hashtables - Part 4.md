
# ArrayList vs Hashtable in C# (.NET Collections)

## Overview

This topic explains why **Hashtable** was introduced, the limitations of **ArrayList**, how Hashtable solves those limitations, and when to use each collection.

> **Note:** In modern .NET, `Dictionary<TKey,TValue>` is generally preferred over `Hashtable`, but understanding `Hashtable` is still valuable for interviews and legacy code.

---

# Review: What is ArrayList?

`ArrayList` is a dynamic collection from `System.Collections`.

```csharp
using System.Collections;

ArrayList list = new ArrayList();
```

Advantages:

- Dynamic size
- Easy Add/Remove methods
- Can store any object

---

# Limitation of ArrayList

The biggest limitation is **lookup by index only**.

Example:

```csharp
Customer customer = (Customer)list[0];
```

Here, `0` is an internally generated index.

In real-world applications, we rarely search by index.

Instead, we search using:

- Customer ID
- Employee ID
- Product Code
- Country Code

---

# Customer Example

```csharp
public class Customer
{
    public int CustomerCode { get; set; }
    public string CustomerName { get; set; }
}
```

Create object:

```csharp
Customer customer = new Customer();

customer.CustomerCode = 1001;
customer.CustomerName = "Shiv";

ArrayList list = new ArrayList();

list.Add(customer);
```

Retrieve:

```csharp
Customer obj = (Customer)list[0];
```

The only way is by index.

---

# What is Hashtable?

`Hashtable` stores data as **Key-Value pairs**.

```csharp
Hashtable table = new Hashtable();
```

Instead of using an index, you supply your own key.

---

# Adding Data

```csharp
Hashtable table = new Hashtable();

table.Add(1001, customer);
```

Here:

- Key = 1001
- Value = Customer object

---

# Retrieving Data

```csharp
Customer obj = (Customer)table[1001];

Console.WriteLine(obj.CustomerName);
```

No need to know the internal index.

---

# How Hashtable Works

Internally:

1. Accepts the key.
2. Converts it into a hash value.
3. Stores the value using that hash.
4. Uses the same hash for fast lookup.

```text
Key (1001)
      │
      ▼
 Hash Function
      │
      ▼
 Hash Code
      │
      ▼
 Hashtable Bucket
```

---

# Why Hashtable Is Slightly Slower

Compared to `ArrayList`, Hashtable performs an extra operation:

- Convert the key into a hash code.

This additional work introduces a small overhead.

---

# ArrayList vs Hashtable

| Feature | ArrayList | Hashtable |
|---------|-----------|-----------|
| Storage | Sequential | Key-Value |
| Lookup | Index | Key |
| Dynamic Size | Yes | Yes |
| Search | By index | By key |
| Performance | Faster iteration | Slightly slower (hash calculation) |
| Best Use | Sequential processing | Fast key lookup |

---

# When to Use ArrayList

Choose ArrayList when:

- You only need sequential access.
- You iterate through all items.
- You don't need custom keys.

---

# When to Use Hashtable

Choose Hashtable when:

- You need lookup by ID or key.
- Fast retrieval by key is important.
- The key uniquely identifies each object.

Examples:

- Customer lookup
- Product catalog
- Employee records
- Configuration settings

---

# Modern .NET Recommendation

Instead of `Hashtable`, use the generic `Dictionary<TKey,TValue>`.

```csharp
Dictionary<int, Customer> customers = new();

customers.Add(1001, customer);

Customer result = customers[1001];
```

Benefits:

- Type-safe
- Better performance
- No casting required
- Generic

---

# Hashtable vs Dictionary

| Hashtable | Dictionary<TKey,TValue> |
|------------|-------------------------|
| Non-generic | Generic |
| Uses object | Strongly typed |
| Requires casting | No casting |
| Legacy collection | Modern collection |

---

# Interview Questions

## What is the biggest limitation of ArrayList?

Items are retrieved primarily by index rather than a user-defined key.

---

## What problem does Hashtable solve?

It allows efficient retrieval using a custom key.

---

## Why is Hashtable slower than ArrayList?

Because it computes a hash code for the supplied key before storing or retrieving an item.

---

## Which collection is recommended in modern C#?

`Dictionary<TKey,TValue>`.

---

# Key Takeaways

- ArrayList is suitable for sequential collections.
- Hashtable stores data as key-value pairs.
- Hashtable enables lookup using custom keys.
- Hash calculation introduces a small overhead.
- For modern .NET applications, prefer `Dictionary<TKey,TValue>` over `Hashtable`.
