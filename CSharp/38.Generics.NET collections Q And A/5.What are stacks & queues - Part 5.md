
# Queue vs Stack in C# (.NET Collections)

## Overview

After learning about **Array**, **ArrayList**, and **Hashtable**, the next category of .NET collections is **Prioritized Collections**.

Unlike index-based or key-value collections, these collections retrieve data in a predefined order.

The two important prioritized collections are:

- **Queue** (FIFO - First In First Out)
- **Stack** (LIFO - Last In First Out)

> **Modern Note:** In current .NET development, use the generic versions `Queue<T>` and `Stack<T>` instead of the non-generic collections whenever possible.

---

# Queue (FIFO)

## What is a Queue?

A Queue follows the **FIFO (First In First Out)** principle.

The first item inserted is the first item removed.

```
Enqueue Order

1
2
3

Dequeue Order

1
2
3
```

---

# Queue Namespace

```csharp
using System.Collections;
```

---

# Create a Queue

```csharp
Queue objQueue = new Queue();
```

---

# Add Items (Enqueue)

```csharp
objQueue.Enqueue(textBox1.Text);
```

`Enqueue()` inserts an item at the end of the queue.

---

# Retrieve Items (Dequeue)

```csharp
textBox1.Text = objQueue.Dequeue().ToString();
```

`Dequeue()` removes and returns the first item.

---

# Complete Queue Example

```csharp
Queue queue = new Queue();

queue.Enqueue("1");
queue.Enqueue("2");
queue.Enqueue("3");

Console.WriteLine(queue.Dequeue()); // 1
Console.WriteLine(queue.Dequeue()); // 2
Console.WriteLine(queue.Dequeue()); // 3
```

Output

```
1
2
3
```

---

# Queue Operations

| Method | Description |
|---------|-------------|
| Enqueue() | Add item |
| Dequeue() | Remove first item |
| Peek() | View first item without removing |
| Clear() | Remove all items |
| Count | Number of elements |

---

# Queue Use Cases

Queues are ideal for:

- Printer queue
- Ticket booking systems
- Call center systems
- Task scheduling
- Background job processing
- Message queues

---

# Stack (LIFO)

## What is a Stack?

A Stack follows the **LIFO (Last In First Out)** principle.

The last item inserted is the first item removed.

```
Push Order

1
2
3

Pop Order

3
2
1
```

---

# Create a Stack

```csharp
Stack objStack = new Stack();
```

---

# Add Items (Push)

```csharp
objStack.Push(textBox1.Text);
```

`Push()` inserts an item on top of the stack.

---

# Retrieve Items (Pop)

```csharp
textBox1.Text = objStack.Pop().ToString();
```

`Pop()` removes and returns the last inserted item.

---

# Complete Stack Example

```csharp
Stack stack = new Stack();

stack.Push("1");
stack.Push("2");
stack.Push("3");

Console.WriteLine(stack.Pop()); // 3
Console.WriteLine(stack.Pop()); // 2
Console.WriteLine(stack.Pop()); // 1
```

Output

```
3
2
1
```

---

# Stack Operations

| Method | Description |
|---------|-------------|
| Push() | Add item |
| Pop() | Remove top item |
| Peek() | View top item |
| Clear() | Remove all items |
| Count | Number of elements |

---

# Stack Use Cases

Stacks are commonly used in:

- Browser Back button
- Undo / Redo
- Function Call Stack
- Expression Evaluation
- Parentheses Matching
- DFS (Depth First Search)

---

# Queue vs Stack

| Feature | Queue | Stack |
|---------|-------|-------|
| Principle | FIFO | LIFO |
| Insert | Enqueue() | Push() |
| Remove | Dequeue() | Pop() |
| Peek | Peek() | Peek() |
| Real-world Example | Printer Queue | Browser Back |

---

# Queue vs Stack Flow

```
Queue (FIFO)

Enqueue
1 → 2 → 3

Dequeue
1 → 2 → 3
```

```
Stack (LIFO)

Push
1
2
3

Pop
3
2
1
```

---

# Modern .NET Recommendation

Instead of:

```csharp
Queue queue = new Queue();
Stack stack = new Stack();
```

Prefer:

```csharp
Queue<string> queue = new Queue<string>();

queue.Enqueue("A");
```

```csharp
Stack<string> stack = new Stack<string>();

stack.Push("A");
```

Advantages:

- Type-safe
- Better performance
- No casting
- No boxing/unboxing

# Interview Questions

## What is FIFO?

**First In First Out**

Implemented using **Queue**.

---

## What is LIFO?

**Last In First Out**

Implemented using **Stack**.

---

## Difference between Enqueue and Dequeue?

- **Enqueue()** adds an item.
- **Dequeue()** removes the first item.

---

## Difference between Push and Pop?

- **Push()** adds an item.
- **Pop()** removes the last inserted item.

---

## What does Peek() do?

Returns the next item without removing it from the collection.

---

## Which generic collections are recommended today?

- `Queue<T>`
- `Stack<T>`

---

# Key Takeaways

- Queue implements **FIFO**.
- Stack implements **LIFO**.
- Queue uses **Enqueue()** and **Dequeue()**.
- Stack uses **Push()** and **Pop()**.
- Both provide a `Peek()` method.
- For modern C#, always prefer **Queue<T>** and **Stack<T>** over their non-generic counterparts.
