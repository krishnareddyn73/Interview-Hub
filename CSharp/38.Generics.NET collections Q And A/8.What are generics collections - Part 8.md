
# Generic Collections in C# (.NET Collections)

## Overview

In the previous topic, we learned that **Generics** separate the **data type** from the **business logic**.

The same concept is applied to .NET collections, resulting in **Generic Collections**.

Generic collections combine the advantages of:

- Strong typing
- Dynamic resizing
- Better performance
- Code reusability

Unlike older collections (`ArrayList`, `Hashtable`), generic collections avoid boxing/unboxing and provide compile-time type safety.

---

# Generic Collections Namespace

All generic collections are available in:

```csharp
using System.Collections.Generic;
```

---

# Why Generic Collections?

Traditional collections have limitations:

| Collection | Limitation |
|------------|------------|
| Array | Fixed size |
| ArrayList | Stores objects (boxing/unboxing) |
| Hashtable | Stores objects (boxing/unboxing) |

Generic collections solve these problems by combining **strong typing** with **dynamic behavior**.

---

# Mapping Non-Generic to Generic Collections

| Non-Generic | Generic Equivalent |
|--------------|-------------------|
| ArrayList | List<T> |
| Hashtable | Dictionary<TKey, TValue> |
| Stack | Stack<T> |
| Queue | Queue<T> |

---

# 1. List<T>

`List<T>` is the generic version of `ArrayList`.

## Create a List

```csharp
List<int> numbers = new List<int>();
```

---

## Add Items

```csharp
numbers.Add(10);
numbers.Add(20);
numbers.Add(30);
```

Only integers are allowed.

---

## Retrieve by Index

```csharp
int value = numbers[0];
```

Like `ArrayList`, `List<T>` is an **index-based collection**.

---

## Different Data Types

```csharp
List<string> names = new List<string>();

names.Add("Shiv");
```

```csharp
List<bool> flags = new List<bool>();

flags.Add(true);
```

---

# 2. Dictionary<TKey, TValue>

`Dictionary<TKey, TValue>` is the generic version of `Hashtable`.

It stores data as **Key-Value pairs**.

---

## Create Dictionary

```csharp
Dictionary<int, string> students = new Dictionary<int, string>();
```

Here:

- Key = int
- Value = string

---

## Add Data

```csharp
students.Add(1, "Shiv");
students.Add(2, "Raj");
```

---

## Retrieve Data

```csharp
string name = students[1];
```

No casting required.

---

# 3. Stack<T>

Generic version of `Stack`.

Follows **LIFO (Last In First Out)**.

---

## Create Stack

```csharp
Stack<int> stack = new Stack<int>();
```

---

## Push Items

```csharp
stack.Push(10);
stack.Push(20);
stack.Push(30);
```

---

## Pop Items

```csharp
int value = stack.Pop();
```

Returns:

```
30
```

The return type is strongly typed.

---

# 4. Queue<T>

Generic version of `Queue`.

Follows **FIFO (First In First Out)**.

---

## Create Queue

```csharp
Queue<int> queue = new Queue<int>();
```

---

## Enqueue

```csharp
queue.Enqueue(10);
queue.Enqueue(20);
queue.Enqueue(30);
```

---

## Dequeue

```csharp
int value = queue.Dequeue();
```

Returns:

```
10
```

---

# Using Custom Classes

Generic collections can store your own classes.

Example:

```csharp
public class Customer
{
    public int CustomerCode { get; set; }
}
```

Create a strongly typed list:

```csharp
List<Customer> customers = new List<Customer>();
```

Add data:

```csharp
customers.Add(new Customer
{
    CustomerCode = 1001
});
```

Only `Customer` objects can be added.

---

# Benefits of Generic Collections

## 1. Strong Typing

Compile-time type checking.

---

## 2. No Boxing / Unboxing

Value types are stored directly.

---

## 3. Better Performance

Faster than older object-based collections.

---

## 4. IntelliSense Support

IDE provides accurate suggestions for the specified type.

---

## 5. No Casting

```csharp
Customer customer = customers[0];
```

No explicit cast is needed.

---

# Comparison

| Feature | Non-Generic | Generic |
|----------|-------------|----------|
| Type Safety | No | Yes |
| Boxing/Unboxing | Yes | No |
| Performance | Lower | Higher |
| Casting Required | Yes | No |
| Compile-Time Checking | No | Yes |

# Common Generic Collections

| Collection | Purpose |
|------------|---------|
| List<T> | Dynamic index-based collection |
| Dictionary<TKey, TValue> | Key-value lookup |
| Queue<T> | FIFO processing |
| Stack<T> | LIFO processing |
| HashSet<T> | Unique values |
| LinkedList<T> | Linked node collection |

# Key Takeaways

- Generic collections are built using the concept of generics.
- `List<T>` replaces `ArrayList`.
- `Dictionary<TKey, TValue>` replaces `Hashtable`.
- `Queue<T>` and `Stack<T>` are generic versions of their non-generic counterparts.
- Generic collections provide better performance, type safety, and maintainability.
- Modern .NET applications should prefer generic collections over older non-generic collections.
