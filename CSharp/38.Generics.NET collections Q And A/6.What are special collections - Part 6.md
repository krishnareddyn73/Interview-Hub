
# Specialized Collections in C# (.NET Collections)

## Overview

After learning about:

- Arrays
- ArrayList
- Hashtable
- Queue
- Stack

the next category of .NET collections is **Specialized Collections**.

Specialized collections are designed for **specific use cases** rather than general-purpose data storage.

> **Important:** In modern .NET, generic collections (`List<T>`, `Dictionary<TKey,TValue>`, etc.) are generally preferred over specialized collections because they are type-safe and typically provide better performance.

---

# Namespace

All specialized collections belong to:

```csharp
using System.Collections.Specialized;
```

---

# Important Specialized Collections

The most commonly discussed specialized collections are:

1. CollectionUtil
2. ListDictionary
3. HybridDictionary
4. StringCollection

---

# 1. CollectionUtil

## Purpose

`CollectionUtil` helps create **case-insensitive collections**.

For example:

- "Shiv"
- "shiv"
- "SHIV"

are treated as the same key.

---

## Create a Case-Insensitive Hashtable

```csharp
using System.Collections.Specialized;

Hashtable table = CollectionUtil.CreateCaseInsensitiveHashtable();
```

---

## Example

```csharp
table["Shiv"] = "Developer";

Console.WriteLine(table["shiv"]);
Console.WriteLine(table["SHIV"]);
Console.WriteLine(table["Shiv"]);
```

Output

```
Developer
Developer
Developer
```

---

## When to Use

Useful when users may enter keys with different letter casing.

Examples:

- Username lookup
- Country names
- Search functionality

---

# 2. ListDictionary

## Purpose

Optimized for **small collections**.

It performs well when there are only a few elements (typically around 10 or fewer).

---

## Create ListDictionary

```csharp
ListDictionary list = new ListDictionary();

list.Add("Sun", "Sunday");
list.Add("Mon", "Monday");
```

---

## Best Use Cases

- Days of the week
- Months of the year
- Small configuration settings
- Small lookup tables

---

# 3. HybridDictionary

## Purpose

Combines the benefits of **ListDictionary** and **Hashtable**.

Internally:

- Small collections → behaves like `ListDictionary`
- Larger collections → automatically switches to `Hashtable`

This provides good performance across varying collection sizes.

---

## Create HybridDictionary

```csharp
HybridDictionary dictionary = new HybridDictionary();

dictionary.Add("A", "Apple");
dictionary.Add("B", "Ball");
```

---

## Internal Working

```
Small Collection
       │
       ▼
ListDictionary

As size increases

       ▼

Hashtable
```

The transition is automatic.

---

## Advantages

- Good performance for both small and large collections
- No need to decide between `ListDictionary` and `Hashtable`

---

# 4. StringCollection

## Purpose

Stores **only string values**.

Unlike `ArrayList`, it does not allow mixed data types.

---

## Create StringCollection

```csharp
StringCollection names = new StringCollection();

names.Add("India");
names.Add("USA");
names.Add("Japan");
```

---

## Advantages

- Stores only strings
- Simpler API
- Prevents accidental insertion of other object types

---

# Comparison of Specialized Collections

| Collection | Purpose | Best Use |
|------------|---------|----------|
| CollectionUtil | Case-insensitive collections | Username or key lookup |
| ListDictionary | Small collections | Fewer than ~10 items |
| HybridDictionary | Automatically switches implementation | Unknown collection size |
| StringCollection | String-only collection | Text collections |

---

# Modern .NET Recommendation

Most specialized collections have modern generic alternatives.

| Specialized Collection | Recommended Alternative |
|------------------------|-------------------------|
| Hashtable | Dictionary<TKey,TValue> |
| ArrayList | List<T> |
| StringCollection | List<string> |
| ListDictionary | Dictionary<TKey,TValue> |
| HybridDictionary | Dictionary<TKey,TValue> |

Generic collections provide:

- Type safety
- Better performance
- No boxing/unboxing
- Cleaner code

# Interview Questions

## What are specialized collections?

Collections designed to solve specific problems rather than general-purpose data storage.

---

## Which namespace contains specialized collections?

```csharp
System.Collections.Specialized
```

---

## What does CollectionUtil do?

Creates case-insensitive collections such as Hashtables and SortedLists.

---

## When should ListDictionary be used?

When working with a very small number of items.

---

## What is HybridDictionary?

A dictionary that automatically uses `ListDictionary` for small collections and switches to `Hashtable` as the collection grows.

---

## What does StringCollection store?

Only string values.

---

## Are specialized collections recommended for modern .NET?

Generally no. Prefer generic collections such as `List<T>` and `Dictionary<TKey,TValue>` unless you are maintaining legacy code or need a specialized behavior.

---

# Key Takeaways

- Specialized collections are designed for specific scenarios.
- `CollectionUtil` creates case-insensitive collections.
- `ListDictionary` is optimized for small collections.
- `HybridDictionary` combines the advantages of `ListDictionary` and `Hashtable`.
- `StringCollection` stores only strings.
- Modern .NET development favors generic collections over specialized collections.
