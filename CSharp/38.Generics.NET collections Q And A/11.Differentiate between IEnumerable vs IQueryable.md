
# IEnumerable vs IQueryable in C# (Entity Framework)

## Overview

`IEnumerable` and `IQueryable` are both important interfaces used for querying collections in .NET.

> **Key Facts**
>
> - Both are collection interfaces.
> - `IQueryable` inherits from `IEnumerable`.
> - Everything `IEnumerable` can do, `IQueryable` can also do.
> - `IQueryable` adds the ability to translate queries to a remote data source (such as SQL).

---

# Interface Relationship

```text
IEnumerable
      ▲
      │
IQueryable
```

Since `IQueryable` inherits from `IEnumerable`, it supports enumeration and adds query translation capabilities.

---

# The Biggest Difference

The most important difference is **where the filtering (or query) is executed**.

| Interface | Filter Executes |
|-----------|-----------------|
| IEnumerable | Client (Memory) |
| IQueryable | Database (SQL Server) |

---

# IEnumerable Example

```csharp
IEnumerable<Employee> employees = context.Employees;

var result = employees.Where(e => e.EmpId == 2);
```

### What Happens?

1. All employee records are fetched from the database.
2. Data is loaded into application memory.
3. `Where()` filter executes in memory.

Generated SQL:

```sql
SELECT * FROM Employees;
```

Filtering happens **after** data reaches the application.

---

# IQueryable Example

```csharp
IQueryable<Employee> employees = context.Employees;

var result = employees.Where(e => e.EmpId == 2);
```

### What Happens?

1. LINQ expression is translated into SQL.
2. SQL Server applies the filter.
3. Only matching records are returned.

Generated SQL:

```sql
SELECT *
FROM Employees
WHERE EmpId = 2;
```

Only the required data travels across the network.

---

# Data Flow Comparison

## IEnumerable

```text
SQL Server
     │
     ▼
All Records
     │
     ▼
Application Memory
     │
     ▼
Where()
```

---

## IQueryable

```text
SQL Server
     │
WHERE EmpId = 2
     │
     ▼
Matching Records Only
     │
     ▼
Application
```

---

# Why IQueryable Is Faster for Databases

Because:

- Less network traffic
- Database performs filtering
- Database indexes are used
- Reduced memory usage
- Better scalability

---

# Why IEnumerable Is Good for In-Memory Data

Use `IEnumerable` when the collection already exists in memory.

Examples:

```csharp
List<Employee>

Employee[]

HashSet<Employee>
```

The database is no longer involved.

---

# Comparison Table

| Feature | IEnumerable | IQueryable |
|---------|-------------|------------|
| Namespace | System.Collections | System.Linq |
| Query Execution | In Memory | Database / Remote Provider |
| SQL Translation | No | Yes |
| Uses Expression Trees | No | Yes |
| Best For | In-memory collections | Entity Framework, LINQ to SQL |

---

# When to Use IEnumerable

Use when:

- Data is already in memory.
- Working with arrays or lists.
- No database interaction.
- Querying local collections.

Example:

```csharp
List<Employee> employees = GetEmployees();

var result = employees.Where(e => e.Age > 30);
```

---

# When to Use IQueryable

Use when:

- Entity Framework
- LINQ to SQL
- Remote data source
- SQL Server
- Azure SQL
- Any LINQ provider that translates queries

Example:

```csharp
var result = context.Employees
                    .Where(e => e.Age > 30)
                    .OrderBy(e => e.Name)
                    .Take(10);
```

Entire query is translated to SQL.

---

# Deferred Execution

Both interfaces support deferred execution.

The query is not executed until data is enumerated.

Example:

```csharp
var query = context.Employees.Where(e => e.Age > 30);

// SQL executes here
var list = query.ToList();
```

---

# Modern Entity Framework Recommendation

When working with Entity Framework:

- Build the query using `IQueryable`.
- Materialize it only when required using:

```csharp
ToList()

ToArray()

First()

FirstOrDefault()

Single()

Count()
```

Once materialized, further operations occur in memory.

# Interview Questions

## Does IQueryable inherit from IEnumerable?

Yes.

---

## What is the biggest difference?

`IEnumerable` filters in memory, while `IQueryable` pushes the query to the database.

---

## Which one reduces network traffic?

`IQueryable`.

---

## Which one should be used with Entity Framework?

`IQueryable`.

---

## Which one should be used for List<T> or arrays?

`IEnumerable`.

---

# Key Takeaways

- `IQueryable` extends `IEnumerable`.
- `IEnumerable` executes queries in application memory.
- `IQueryable` translates LINQ into SQL and executes it on the database.
- Use `IEnumerable` for in-memory collections.
- Use `IQueryable` for database-backed collections such as Entity Framework.
- Materialize an `IQueryable` only after applying all required filters and projections.
