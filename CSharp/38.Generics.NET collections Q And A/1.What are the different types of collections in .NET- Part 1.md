# .NET Collections – Overview (Interview Notes)

## What is a Collection?

A **collection** is a group of related records (objects) that can be treated as a single logical unit.

Examples:
- Country Collection → Country Code, Country Name
- Product Collection → Product ID, Product Name
- Employee Collection → Employee ID, Employee Name
- Book Collection → ISBN, Book Name

Instead of managing individual objects separately, collections allow you to manage them together.

---

# Categories of .NET Collections

.NET collections can be broadly divided into **4 categories**:

```text
.NET Collections
│
├── 1. Index-Based Collections
│      ├── Array
│      └── List<T>
│
├── 2. Key-Value Pair Collections
│      ├── Hashtable
│      └── SortedList
│
├── 3. Prioritized Collections
│      ├── Queue<T>
│      └── Stack<T>
│
└── 4. Specialized Collections
       ├── StringCollection
       ├── HybridDictionary
       ├── NameValueCollection
       └── Others
```

# 1. Index-Based Collections

These collections access data using an **index number**.

Whenever you add an item, .NET automatically assigns an internal index.

Example:

| Index | Country |
|------:|---------|
| 0 | India |
| 1 | USA |
| 2 | Nepal |

```csharp
string[] countries =
{
    "India",
    "USA",
    "Nepal"
};

Console.WriteLine(countries[0]); // India
```

Or

```csharp
List<string> countries = new()
{
    "India",
    "USA",
    "Nepal"
};

Console.WriteLine(countries[0]);
```

### Characteristics

- Uses numeric indexes
- Fast random access
- Index generated internally
- Best when position matters

## Array
- Fixed size
- Better performance
- Contiguous memory

```csharp
int[] numbers = { 10, 20, 30 };
```

## List<T>
- Dynamic size
- Automatically grows
- Most commonly used collection

```csharp
List<int> numbers = new();

numbers.Add(10);
numbers.Add(20);
numbers.Add(30);
```

# 2. Key-Value Pair Collections

Access data using a key instead of an index.

| Key | Value |
|------|-------|
| IND | India |
| USA | United States |
| NEP | Nepal |

```csharp
Hashtable countries = new Hashtable();

countries["IND"] = "India";
countries["USA"] = "United States";
countries["NEP"] = "Nepal";

Console.WriteLine(countries["IND"]);
```

Output

```text
India
```

## Why Key-Value Collections?

Useful when you know:
- Employee ID
- Product ID
- Country Code
- Customer ID

### Hashtable
- Fast lookup
- Unordered

### SortedList
- Sorted by key
- Lookup by key

# 3. Prioritized Collections

## Queue (FIFO)

**First In First Out**

Examples:
- Printer queue
- Bank queue
- Ticket booking

```csharp
Queue<string> queue = new();

queue.Enqueue("A");
queue.Enqueue("B");
queue.Enqueue("C");

Console.WriteLine(queue.Dequeue()); // A
```

## Stack (LIFO)

**Last In First Out**

Examples:
- Browser Back
- Undo
- Call Stack

```csharp
Stack<string> stack = new();

stack.Push("A");
stack.Push("B");
stack.Push("C");

Console.WriteLine(stack.Pop()); // C
```

### Queue vs Stack

| Queue | Stack |
|--------|-------|
| FIFO | LIFO |
| Enqueue | Push |
| Dequeue | Pop |

# 4. Specialized Collections

# Key Takeaways

- Collections store multiple related objects.
- Arrays and List<T> are index-based.
- Hashtable and SortedList are key-value collections.
- Queue follows FIFO.
- Stack follows LIFO.
- Specialized collections solve specific problems.
- Modern .NET favors generic collections such as `List<T>`, `Dictionary<TKey,TValue>`, `Queue<T>`, and `Stack<T>` because they are type-safe and more efficient.
