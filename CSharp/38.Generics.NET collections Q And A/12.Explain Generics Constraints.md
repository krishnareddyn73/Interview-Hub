
# Generic Constraints in C#

## Overview

Generics allow us to **separate business logic from the data type**, making code reusable and type-safe.

However, allowing **any type** to be used with a generic class is not always desirable.

This is where **Generic Constraints** come into the picture.

Generic constraints allow you to **restrict which types can be used with a generic type parameter**.

---

# Why Do We Need Generic Constraints?

Consider a generic Data Access Layer (DAL).

```csharp
public class DataAccess<T>
{
    public void Save(T entity)
    {
        // Save entity to database
    }
}
```

This class should work with business objects such as:

- Customer
- Supplier
- Product

But it does **not** make sense to use:

```csharp
DataAccess<int>

DataAccess<double>

DataAccess<bool>
```

These are value types and do not represent business entities.

Generic constraints prevent such invalid usage.

---

# Generic Constraint Syntax

Constraints are specified using the `where` keyword.

```csharp
public class DataAccess<T>
    where T : class
{
}
```

Here:

- `T` must be a **reference type**.
- Value types such as `int` or `bool` are not allowed.

---

# Valid Example

```csharp
public class Customer
{
}

DataAccess<Customer> customerDAL = new DataAccess<Customer>();
```

This compiles successfully.

---

# Invalid Example

```csharp
DataAccess<int> dal = new DataAccess<int>();
```

Compile-time error because `int` is not a reference type.

---

# Common Generic Constraints

## 1. class

Reference types only.

```csharp
where T : class
```

---

## 2. struct

Value types only.

```csharp
where T : struct
```

---

## 3. new()

Type must have a public parameterless constructor.

```csharp
where T : new()
```

Example:

```csharp
public T Create()
{
    return new T();
}
```

---

## 4. Base Class Constraint

```csharp
where T : Person
```

Only `Person` or derived classes are allowed.

---

## 5. Interface Constraint

```csharp
where T : IDisposable
```

Only types implementing `IDisposable` are permitted.

---

## 6. Multiple Constraints

```csharp
where T : Person, IDisposable, new()
```

Meaning:

- Inherits from `Person`
- Implements `IDisposable`
- Has a parameterless constructor

---

# Real-World Example

```csharp
public class Repository<T>
    where T : class
{
    public void Save(T entity)
    {
        // Save entity
    }
}
```

Usage:

```csharp
Repository<Customer> repository = new Repository<Customer>();
```

Invalid:

```csharp
Repository<int> repository = new Repository<int>();
```

---

# Benefits of Generic Constraints

- Type safety
- Prevent invalid generic usage
- Compile-time validation
- Better IntelliSense
- Cleaner and more maintainable code

---

# Common Constraint Summary

| Constraint | Description |
|------------|-------------|
| class | Reference type |
| struct | Value type |
| new() | Public parameterless constructor |
| BaseClass | Must inherit from specified class |
| Interface | Must implement specified interface |

# Interview Questions

## What are Generic Constraints?

Rules that restrict the types allowed for a generic parameter.

---

## Which keyword is used to define constraints?

```csharp
where
```

---

## What does `where T : class` mean?

`T` must be a reference type.

---

## What does `where T : new()` mean?

`T` must have a public parameterless constructor.

---

## Can multiple constraints be applied?

Yes.

Example:

```csharp
where T : Person, IDisposable, new()
```

---

# Key Takeaways

- Generic constraints restrict the types that can be used with generics.
- Constraints are declared using the `where` keyword.
- `class` prevents value types from being used.
- `new()` allows generic code to create instances of `T`.
- Constraints improve compile-time safety and make generic code more reliable.
