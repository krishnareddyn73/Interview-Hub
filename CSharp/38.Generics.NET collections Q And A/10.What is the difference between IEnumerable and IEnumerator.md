
# IEnumerable vs IEnumerator in C#

## Overview

`IEnumerable` and `IEnumerator` are closely related interfaces used for traversing collections.

Many developers know how to use them but are confused about their differences.

> **Key Idea:** `IEnumerable` is a higher-level abstraction that internally uses `IEnumerator`.

---

# Namespace

```csharp
using System.Collections.Generic;
```

---

# Sample Collection

```csharp
List<int> years = new List<int>
{
    1991,
    1992,
    2001,
    2002,
    2003
};
```

---

# Using IEnumerable

```csharp
IEnumerable<int> data = years;

foreach (int year in data)
{
    Console.WriteLine(year);
}
```

Output

```
1991
1992
2001
2002
2003
```

`foreach` automatically obtains an `IEnumerator` by calling `GetEnumerator()`.

---

# Using IEnumerator

```csharp
IEnumerator<int> iterator = years.GetEnumerator();

while (iterator.MoveNext())
{
    Console.WriteLine(iterator.Current);
}
```

Output

```
1991
1992
2001
2002
2003
```

---

# Relationship

```
Collection
     │
     ▼
IEnumerable
     │
 GetEnumerator()
     ▼
IEnumerator
```

`IEnumerable` is essentially a convenient wrapper around `IEnumerator`.

---

# Main Difference

## IEnumerable

- Represents a collection.
- Creates a new enumerator.
- Does **not** expose the current iteration state directly.

---

## IEnumerator

- Represents the iterator itself.
- Maintains the current cursor position.
- Exposes:
  - `MoveNext()`
  - `Current`
  - `Reset()` (rarely used)

---

# Why IEnumerator Matters

Because it remembers where iteration currently is.

Example:

```text
1991
1992
2001  <-- Current Position
2002
2003
```

You can pass the same `IEnumerator` to another method and continue from the current position.

```csharp
void ProcessRemaining(IEnumerator<int> iterator)
{
    while (iterator.MoveNext())
    {
        Console.WriteLine(iterator.Current);
    }
}
```

Iteration resumes instead of starting over.

---

# Why IEnumerable Cannot Do This

Each `foreach` obtains a **new** enumerator.

```csharp
foreach (var item in data)
{
}
```

A second `foreach` starts again from the beginning.

The iteration state is not shared.

---

# Feature Comparison

| Feature | IEnumerable | IEnumerator |
|---------|-------------|-------------|
| Represents | Collection | Iterator |
| Used by | foreach | Manual iteration |
| Keeps current position | No | Yes |
| MoveNext() | No | Yes |
| Current | No | Yes |
| GetEnumerator() | Yes | No |

---

# When to Use IEnumerable

Use when:

- Exposing read-only collections
- Returning sequences
- Using `foreach`
- LINQ queries

---

# When to Use IEnumerator

Use when:

- Manual control of iteration is required
- You need to pause/resume iteration
- Passing the current iterator between methods

---

# Modern .NET Notes

Prefer the generic interfaces:

```csharp
IEnumerable<T>

IEnumerator<T>
```

instead of the non-generic versions.

---

# Interview Questions

## What is the relationship between IEnumerable and IEnumerator?

`IEnumerable` creates and returns an `IEnumerator` through `GetEnumerator()`.

---

## Which interface is used by foreach?

`IEnumerable`.

---

## Which interface maintains the current cursor position?

`IEnumerator`.

---

## Why is IEnumerator called a stateful iterator?

Because it remembers the current element while iterating.

---

# Key Takeaways

- `IEnumerable` represents a sequence of items.
- `IEnumerator` performs the actual iteration.
- `foreach` internally uses `IEnumerator`.
- `IEnumerator` maintains iteration state, while `IEnumerable` does not.
- Use `IEnumerable<T>` for exposing collections and `IEnumerator<T>` when manual iteration control is needed.
