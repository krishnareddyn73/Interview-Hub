# Mutex, Semaphore & SemaphoreSlim in C# - Interview Notes

## Why Do We Need Them?

In a multi-threaded application, multiple threads may try to access the same resource simultaneously, causing:
- Race conditions
- Data corruption
- Unexpected results
- Application crashes

## Synchronization Mechanisms

| Mechanism | Used For | Scope |
|-----------|-----------|--------|
| lock | Thread safety | In-Process |
| Monitor | Thread safety | In-Process |
| Mutex | Thread safety | Cross-Process |
| Semaphore | Limit concurrent access | Cross-Process |
| SemaphoreSlim | Limit concurrent access | In-Process |

---

# Lock and Monitor

- Allow only one thread at a time.
- Used within the same application.

```csharp
lock(this)
{
    // Critical section
}
```

Monitor equivalent:

```csharp
Monitor.Enter(this);
try
{
    // Critical section
}
finally
{
    Monitor.Exit(this);
}
```

---

# Mutex

Used when multiple applications/processes need synchronization.

```csharp
using var m1 = new Mutex(false, "ConsoleApplication6");

if (!m1.WaitOne(5000))
{
    Console.WriteLine("Already an instance is running");
    return;
}
```

Mutex = Cross Process Lock

---

# Semaphore

Allows a limited number of threads simultaneously.

```csharp
Semaphore s1 = new Semaphore(3, 3, "ConsoleApplication6");

s1.WaitOne();

// Work

s1.Release();
```

Example:
- Capacity = 3
- First 3 threads enter
- 4th thread waits

---

# SemaphoreSlim

Lightweight version of Semaphore.

```csharp
SemaphoreSlim sem = new SemaphoreSlim(3);

await sem.WaitAsync();

// Work

sem.Release();
```

Advantages:
- Faster
- Lower memory usage
- Async friendly

---

# Comparison

| Feature | Lock | Monitor | Mutex | Semaphore | SemaphoreSlim |
|----------|------|----------|--------|-----------|---------------|
| One Thread | Yes | Yes | Yes | No | No |
| Multiple Threads | No | No | No | Yes | Yes |
| Cross Process | No | No | Yes | Yes | No |
| Lightweight | Yes | Yes | No | No | Yes |
| Async Friendly | No | No | No | Limited | Yes |

---

# Interview Summary

- Lock/Monitor → One thread, In-Process
- Mutex → One thread, Cross-Process
- Semaphore → Multiple threads, Cross-Process
- SemaphoreSlim → Multiple threads, In-Process

### Rule

- Same Application + One Thread → Lock
- Multiple Applications + One Thread → Mutex
- Same Application + Limited Threads → SemaphoreSlim
- Multiple Applications + Limited Threads → Semaphore
