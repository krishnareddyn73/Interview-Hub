# Task Parallel Library (TPL) in C# - Interview Notes

## What is TPL?

Task Parallel Library (TPL) is a higher-level abstraction over threading provided by:

```csharp
System.Threading.Tasks
```

TPL helps utilize multiple CPU cores efficiently and simplifies parallel programming.

---

# Problem with Traditional Threads

Example:

```csharp
Thread t1 = new Thread(RunMillionIterations);
t1.Start();
```

Even though the code runs in a separate thread, it may execute on only one CPU core.

### Result

- Context switching occurs.
- CPU cores may not be fully utilized.
- Developer must manage work distribution manually.

---

# Threading vs Parallelism

## Threading

```text
Thread 1
Thread 2
```

The operating system performs:

```text
Time Slicing
Context Switching
```

Threads share processor time.

---

## True Parallelism

```text
Core 1 -> Task A
Core 2 -> Task B
Core 3 -> Task C
Core 4 -> Task D
```

Multiple cores execute work simultaneously.

---

# Why TPL Was Introduced

Modern CPUs contain:

- 2 cores
- 4 cores
- 8 cores
- 16+ cores

Applications should utilize all available hardware resources.

TPL automatically:

- Splits work
- Schedules tasks
- Uses available cores
- Aggregates results

---

# Namespace

```csharp
using System.Threading.Tasks;
```

# Parallel.For Example

Traditional Loop:

```csharp
for(int i = 0; i < 1000000; i++)
{
    // Work
}
```

TPL Version:

```csharp
Parallel.For(
    0,
    1000000,
    i =>
    {
        // Work
    });
```

---

# How Parallel.For Works

TPL:

1. Divides work into chunks.
2. Distributes chunks across CPU cores.
3. Executes chunks simultaneously.
4. Combines results.

Developer focuses on business logic.

TPL handles scheduling.

---

# Benefits of TPL

## 1. Multi-Core Optimization

TPL automatically uses available CPU cores.

Benefits:

- Better CPU utilization
- Faster execution
- Improved scalability

---

## 2. Thread Pooling

TPL uses the .NET Thread Pool internally.

Benefits:

- Reuses existing threads
- Reduces thread creation overhead
- Reduces memory usage
- Reduces Garbage Collection pressure

---

# Traditional Thread Creation

```csharp
Thread t =
    new Thread(Method1);

t.Start();
```

Problems:

- Allocates memory
- Creates OS thread
- Requires cleanup
- Increases GC work

```csharp

{

```

Benefits:

- Uses Thread Pool
- Faster

- Better scalability

# Why "Task" Instead of "Thread"?

This is one of the most important interview concepts.

TPL intentionally changed the vocabulary from:

```text
Thread
```

to

```text
Task
```

---

# Thread = How

Thread describes:

```text
How work is executed
```

Example:

```text
Thread
Async Delegate
Thread Pool
```

These are execution mechanisms.

---

# Task = What

Task describes:

```text
What work should be done
```

Example:

```csharp
CalculateSalary()
ProcessOrders()
GenerateReport()
```

These are tasks.

---

# TPL Philosophy

Traditional Thinking:

```text
Create Thread
Execute Method
Manage Thread
```

TPL Thinking:

```text
Create Task
Let TPL Decide Execution Strategy
```

You focus only on:

```text
What to execute
```

TPL handles:

```text
How to execute
```

---

# Abstraction

Task is an abstraction.

A task may be executed by:

- Thread Pool Thread
- Multiple Threads
- Asynchronous Operations
- Parallel Processing

Task does not necessarily mean:

```text
One Task = One Thread
```

This is a common interview question.

---

# Task is Bigger than Thread

Think of it like:

```text
Task
 └── Thread
 └── Thread Pool
 └── Async Operation
 └── Parallel Execution
```

Thread is only one implementation mechanism.

Task is the higher-level concept.
