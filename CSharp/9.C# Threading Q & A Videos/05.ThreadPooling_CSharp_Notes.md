# Thread Pooling in C# - Interview Notes

## What is Thread Pooling?

Thread Pooling is a mechanism where threads are reused instead of being created and destroyed repeatedly.

### Normal Thread Lifecycle

1. Request arrives.
2. Thread object created.
3. Resources allocated.
4. Task executed.
5. Thread becomes eligible for Garbage Collection.

This process repeats for every request.

---

## Problem with Normal Threads

Creating threads repeatedly causes:

- Memory allocation overhead
- CPU overhead
- Increased Garbage Collection
- Performance degradation

---

## Thread Pool Solution

Instead of destroying threads after execution:

1. Create thread once.
2. Execute task.
3. Return thread to pool.
4. Reuse thread for future requests.

### Flow

Request → Thread Pool → Available Thread → Execute Task → Return to Pool

---

## Advantages of Thread Pool

### 1. Reuse Threads

Thread creation and resource allocation happen only once.

### 2. Limit Thread Count

Prevents creating thousands of threads.

Too many threads can:

- Increase context switching
- Increase OS overhead
- Reduce performance

---

## Using Normal Thread

```csharp
Thread t = new Thread(Function1);
t.Start();
```

Creates a new thread every time.

---

## Using Thread Pool

```csharp
ThreadPool.QueueUserWorkItem(
    new WaitCallback(Function1));
```

Function Signature:

```csharp
public static void Function1(object state)
{
    Console.WriteLine("Called");
}
```

---

## Example

```csharp
ThreadPool.QueueUserWorkItem(
    new WaitCallback(Function1));

Console.ReadLine();
```

The thread is taken from the pool, executes the task, and returns to the pool.

---

## Performance Benchmark

### Without Thread Pool

```csharp
for(int i = 0; i < 10; i++)
{
    Thread t = new Thread(Function1);
    t.Start();
}
```

### With Thread Pool

```csharp
for(int i = 0; i < 10; i++)
{
    ThreadPool.QueueUserWorkItem(
        new WaitCallback(Function1));
}
```


Purpose:

- Trigger JIT Compilation
- Exclude compilation time
- Measure actual execution time

---

## Why Thread Pool is Faster

Because it avoids:

- Thread creation
- Resource allocation
- Frequent garbage collection

The framework reuses existing threads.

---

## Interview Questions

### What is Thread Pool?

A collection of reusable worker threads managed by .NET.

### Why use Thread Pool?

- Better performance
- Reduced memory usage
- Reduced thread creation overhead

### Difference Between Thread and Thread Pool?

| Feature | Thread | Thread Pool |
|----------|---------|-------------|
| Creation | New Thread Every Time | Reused Threads |
| Performance | Slower | Faster |
| Resource Usage | Higher | Lower |
| Managed By | Developer | .NET Runtime |

---

## Modern Alternative

Instead of directly using ThreadPool:

```csharp
Task.Run(() =>
{
    // Work
});
```

Task Parallel Library (TPL) internally uses the Thread Pool.

---

## Quick Summary

- Thread Pool = Collection of reusable threads.
- Improves performance by avoiding thread creation.
- Controls maximum thread count.
- Managed by .NET Runtime.
- Task.Run() uses Thread Pool internally.
- Preferred over creating raw Thread objects in modern .NET.
