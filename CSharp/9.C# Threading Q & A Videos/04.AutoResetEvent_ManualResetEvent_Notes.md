# AutoResetEvent and ManualResetEvent in C#

## Overview

AutoResetEvent and ManualResetEvent are synchronization primitives used for thread signaling.

## AutoResetEvent

- Releases only one waiting thread.
- Automatically resets after releasing a thread.
- Each WaitOne() requires a Set().

Example:

```csharp
static AutoResetEvent objAuto = new AutoResetEvent(false);

objAuto.WaitOne();
objAuto.Set();
```

### AutoResetEvent Flow

Starting -> WaitOne() -> Thread Waits -> Set() -> Thread Continues

## ManualResetEvent

- Releases all waiting threads.
- Remains signaled until Reset() is called.
- One Set() can release multiple WaitOne() calls.

Example:

```csharp
static ManualResetEvent objManual = new ManualResetEvent(false);

objManual.WaitOne();
objManual.Set();
```

## Comparison

| Feature | AutoResetEvent | ManualResetEvent |
|----------|---------------|------------------|
| Releases | One Thread | All Threads |
| Auto Reset | Yes | No |
| Set Required | For every WaitOne | One Set can release many |
| Behaviour | Turnstile Gate | Open Gate |

## Interview Summary

AutoResetEvent
- One signal releases one thread.
- Automatically resets.

ManualResetEvent
- One signal releases all waiting threads.
- Stays open until Reset().

Memory Trick:

AutoResetEvent = Turnstile Gate
ManualResetEvent = Open Gate
