## Synchronous Programming

A synchronous call means the caller waits until the callee finishes.

```text
Caller --> Request --> Callee
           Waiting...
Caller <-- Response <-- Callee
```

Example:

```csharp
BigInteger result = obj.Calculate(100000);
```

---

## Asynchronous Programming

The caller does not wait for the callee.

```text
Caller --> Request --> Callee

Caller continues doing other work

Later... receives result
```

---

## Threading

Thread = Independent path of execution.

```csharp
Thread t1 = new Thread(() =>
{
    obj.Calculate(10000);
});

t1.Start();
```

Threads help implement asynchronous behavior.

---

## Delegates

Delegate = Type-safe function pointer.

Used mainly for:

- Callbacks
- Communication between caller and callee
- Event handling

Example:

```csharp
public delegate void SendResult(BigInteger result);
```

Callback:

```csharp
obj.WhereToSend = DisplayFactorial;
```

---

## Multicast Delegates

A multicast delegate can call multiple methods.

```csharp
obj.WhereToSend += DisplayFactorial;
obj.WhereToSend += DisplayFactorialWhenGreaterThan10K;
```

One delegate can notify many subscribers.

---

## Events

Events encapsulate delegates and provide a true Publisher-Subscriber model.

```csharp
public event SendResult WhereToSend;
```

Subscribers can:

```csharp
+= Subscribe
-= Unsubscribe
```

Subscribers cannot:

```csharp
obj.WhereToSend = null;
obj.WhereToSend(100);
```

## Interview One-Liners

### Thread

### Delegate
Type-safe function pointer used for callbacks.

### Multicast Delegate
Delegate capable of invoking multiple methods.

### Event
Restricted multicast delegate implementing Publisher-Subscriber architecture.
