# Async and Await in .NET (Interview Notes)

## What are Async and Await?

`async` and `await` are keywords introduced in .NET Framework 4.5 to simplify asynchronous programming.

### Simple Definition

> Async and Await are code markers that indicate where execution should pause and from where it should resume after an asynchronous task completes.

---

## Why Do We Need Async and Await?

They allow long-running tasks to execute without blocking application flow and provide a clean way to resume execution when the task completes.

---

## Example

```csharp
public static async void CallMethod()
{
    await Task.Factory.StartNew(LongTask);
    Console.WriteLine("New Thread");
}

public static void LongTask()
{
    Thread.Sleep(20000);
}
```

### Execution Flow

1. Execution reaches the `await` statement.
2. The long-running task starts.
3. The method pauses at the await point.
4. The thread is released for other work.
5. When the task completes, execution resumes after the await statement.
6. `Console.WriteLine("New Thread")` executes.

---

## Relationship Between Async and Await

- `await` can only be used inside an `async` method.
- `async` marks a method that contains `await`.
- They are typically used together.

Example:

```csharp
public async Task MyMethod()
{
    await SomeTask();
}
```

---

## Important Points

- `await` does not create a new thread.
- `await` does not block the calling thread.
- Execution resumes from the line immediately after the `await` statement.
- Ideal for I/O operations, API calls, database access, and long-running tasks.

---

## Interview Questions

### What is the purpose of async and await?

They simplify asynchronous programming by allowing code execution to pause and resume without blocking threads.

### Does await create a new thread?

No. It only waits for an asynchronous operation to complete.

### Does await block the calling thread?

No. It pauses the method execution but releases the thread.

### Can we use await without async?

No. A method containing `await` must be marked with `async`.

---

## One-Line Interview Answer

> Async and Await are .NET keywords used to write non-blocking asynchronous code by pausing execution at an await point and automatically resuming it when the asynchronous operation completes.
